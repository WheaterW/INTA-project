(Optional) Enabling IPsec

To enable the VSUP to process IPsec services, you need to enable IPsec.

Prerequisites

  • The corresponding GTL license file has been loaded through the license active file-name command.
  • Before configuring IPsec on the main control board, you do not need to enable IPsec. However, before configuring IPsec on the VSUP, you need to run the vsm on-board-mode disable command in the system view.

After you run the vsm on-board-mode disable command, IPsec cannot be configured on the main control board.

Only the following models support this configuration:

NE40E-M2K

NE40E-M2K-B

Procedure

  1. Run system-view

    The system view is displayed.

  2. Run license

    The license view is displayed.

  3. Run active ipsec bandwidth-enhance value slot slot-id

    The bandwidth resource enhancement function is activated for the VSUP. The default IPsec bandwidth of a board is 0 Gbit/s. If this function is not activated, IPsec cannot be used on the board.

  4. Run commit

    The configuration is committed.

Verifying the Configuration

Run the display ipsec bandwidth [ slot slot-id ] command to check the bandwidth configured in the IPsec license.
