peer default-route-advertise(BGP-VPN-Target address family view)
================================================================

peer default-route-advertise(BGP-VPN-Target address family view)

Function
--------



The **peer default-route-advertise** command configures a device to advertise default routes to its peer.

The **undo peer default-route-advertise** command restores the default configuration.



By default, a device does not advertise any default route to its peer.


Format
------

For CE8851-32CQ8DQ-P, CE8850-HAM, CE8850-SAN, CE8851K, CE8855, CE8851-32CQ4BQ, CE6860-SAN, CE6866, CE6860-HAM, CE6866K, CE6820H, CE6820H-K, CE6820S, CE6863H, CE6863H-K, CE6881H, CE6881H-K, CE6855-48XS8CQ, CE6885, CE6885-T, CE6885-LL (standard forwarding mode), CE6863E-48S8CQ, CE6885-SAN:

**peer** { *ipv4-address* | *ipv6-address* } **default-route-advertise**

**undo peer** { *ipv4-address* | *ipv6-address* } **default-route-advertise**

For CE6885-LL (low latency mode):

**peer** *ipv4-address* **default-route-advertise**

**undo peer** *ipv4-address* **default-route-advertise**


Parameters
----------

| Parameter | Description | Value |
| --- | --- | --- |
| *ipv4-address* | Specifies the IPv4 address of a peer. | The value is in dotted decimal notation. |
| *ipv6-address* | Specifies an IPv6 peer address.  NOTE:  This parameter is supported only on the CE6863H, CE6863H-K, CE6860-SAN, CE6866K, CE6866, CE6860-HAM, CE6855-48XS8CQ, CE6885-SAN, CE8850-SAN, CE8855, CE8851-32CQ4BQ, CE8851K, CE8851-32CQ8DQ-P, CE8850-HAM, CE6881H, CE6881H-K, CE6820H, CE6820H-K, CE6820S, CE6885, CE6885-T, CE6885-LL (standard forwarding mode) and CE6863E-48S8CQ. | The value is a 32-bit hexadecimal string in format X:X:X:X:X:X:X:X. |



Views
-----

BGP-VPN-target address family view


Default Level
-------------

2: Configuration level


Usage Guidelines
----------------

**Usage Scenario**

In an ORF scenario, if a device that does not support ORF exists on the network, you can configure the device to advertise ORF default routes to a peer group in the BGP-VPN-Target address family view to establish an ORF peer relationship.

**Precautions**

This command enables the device to advertise the default route generated by itself, and condition-based matching is not used. Therefore, you are not advised to configure if-match clauses in a route-policy.


Example
-------

# Configure a BGP device to advertise default routes to its peer.
```
<HUAWEI> system-view
[~HUAWEI] bgp 100
[*HUAWEI-bgp] peer 10.1.1.1 as-number 100
[*HUAWEI-bgp] ipv4-family vpn-target
[*HUAWEI-bgp-af-vpn-target] peer 10.1.1.1 enable
[*HUAWEI-bgp-af-vpn-target] peer 10.1.1.1 default-route-advertise

```
