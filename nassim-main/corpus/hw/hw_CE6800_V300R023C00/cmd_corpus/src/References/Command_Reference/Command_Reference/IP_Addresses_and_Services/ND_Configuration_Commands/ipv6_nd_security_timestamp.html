ipv6 nd security timestamp

Function

The ipv6 nd security timestamp command sets timestamp parameters for an ND message.

The undo ipv6 nd security timestamp command restores default timestamp parameters of an ND message.

By default, the maximum difference between the receive time and send time of an ND message is 300 seconds; the maximum difference between the system time of the sender and the system time of the receiver is 1%; the maximum alive time of an ND message is 1 second.

This command is supported only on the CE6863H, CE6863H-K, CE6860-SAN, CE6866K, CE6866, CE6860-HAM, CE6855-48XS8CQ, CE6885-SAN, CE8850-SAN, CE8855, CE8851-32CQ4BQ, CE8851K, CE8851-32CQ8DQ-P, CE8850-HAM, CE6881H, CE6881H-K, CE6820H, CE6820H-K, CE6820S, CE6885, CE6885-T, CE6885-LL (standard forwarding mode) and CE6863E-48S8CQ.

Format

ipv6 nd security timestamp { delta delta-value | drift drift-value | fuzz-factor fuzz-value } *

undo ipv6 nd security timestamp delta

undo ipv6 nd security timestamp drift

undo ipv6 nd security timestamp fuzz-factor

undo ipv6 nd security timestamp { delta delta-value | drift drift-value | fuzz-factor fuzz-value } *

Parameters

Parameter Description Value
delta delta-value

Specifies the maximum difference between the receive time and send time of an ND message.

The value is an integer ranging from 0 to 1000, in seconds.
drift drift-value

Specifies the maximum difference between the system time of the sender and the system time of the receiver.

The value is an integer ranging from 0 to 100.
fuzz-factor fuzz-value

Specifies the maximum alive time of an ND message. If the difference between the receive time and send time of an ND message is larger than delta-value but smaller than fuzz-value, the ND message can still be received by the interface.

The value is an integer ranging from 0 to 1000, in seconds.

Usage Guidelines

Usage Scenario

If an ND message sent to an interface is intercepted by an attacker, the ND message will be delayed to reach the interface. Therefore, to limit the difference between the time when the local end receives an ND message and the time when the peer end sends an ND message, you can run this command to set timestamp parameters. The system calculates the time range based on these parameters. If the difference between the time when the packet is received and the time when the packet is sent exceeds the time range that the interface can receive, the packet is considered invalid and discarded.

  • If the peer end does not establish a neighbor entry with the local end, the time range that can be received is calculated according to the following formula: -delta-value < (RDnew - TSnew) < +delta-value
  • If the peer end has established a neighbor entry with the local end, the time range that can be received is calculated according to the following formula: TSnew + fuzz-value > TSlast + (RDnew - RDlast) x (1 - drift-value) - fuzz-value
  • RDnew: time when the receiver receives the latest SEND message
  • RDlast: time when the receiver receives the last SEND message from the same peer
  • TSnew: indicates the timestamp value in the latest SEND message received by the receiver. The timestamp value is recorded by the sender in the Timestamp option of the SEND message.
  • TSlast: indicates the timestamp value in the last SEND message received and received by the receiver. The timestamp value is recorded by the sender in the Timestamp option of the SEND message.

    For example, if Device A sends an ND message to Device B for the first time and the system time of Device A is 4:00, TSnew is 4:00. When Device B receives the ND message, the system time of Device B is 5:00. Therefore, RDnew is 5:00. If the message is considered secure, Device B records TSlast as 4:00 and RDlast as 5:00.

    When Device A sends an ND message to Device B again, the system time of Device A is 4:05. Therefore, TSnew is 4:05. When Device B receives the ND message, the system time of Device B is 5:05. Therefore, RDnew is 5:05. If the message is considered secure, Device B records TSlast as 4:05 and RDlast as 5:05.

Prerequisites

IPv6 has been enabled on the involved interface using the ipv6 enable command in the interface view.

Follow-up Procedure

Run the ipv6 nd security strict command to enable the strict security mode on the interface.

Example

# Set timestamp parameters for an ND message on 100GE1/0/1 interfaces. The maximum difference between the receive time and send time of an ND message is 10 seconds.
<HUAWEI> system-view
[~HUAWEI] interface 100GE 1/0/1
[~HUAWEI-100GE1/0/1] undo portswitch
[*HUAWEI-100GE1/0/1] ipv6 enable
[*HUAWEI-100GE1/0/1] ipv6 nd security timestamp delta 10
[*HUAWEI-100GE1/0/1] ipv6 nd security strict
