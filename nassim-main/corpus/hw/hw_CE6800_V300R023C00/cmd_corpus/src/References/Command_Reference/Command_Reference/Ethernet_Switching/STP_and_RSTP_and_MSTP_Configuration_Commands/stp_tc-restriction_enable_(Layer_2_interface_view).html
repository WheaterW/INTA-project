stp tc-restriction enable (Layer 2 interface view)

Function

The stp tc-restriction enable command enables TC/TCN packet suppression.

The undo stp tc-restriction enable command disables TC/TCN packet suppression.

By default, TC/TCN packet suppression is disabled.

Format

stp tc-restriction enable

undo stp tc-restriction enable

Parameters

None

Views

Layer 2 100GE interface view,Layer 2 10GE interface view,Layer 2 200GE interface view,25GE-L2 view,400GE-L2 view,Layer 2 50GE interface view,Layer 2 Eth-Trunk interface view,Layer 2 GE interface view,Interface group view

Default Level

2: Configuration level

Usage Guidelines

Usage Scenario

If TC/TCN packets do not need to be flooded, you can run the stp tc-restriction enable command to enable TC/TCN packet suppression. In this way, after receiving TC/TCN packets, an interface neither updates local ARP and MAC entries nor floods the packets to other interfaces on the device.

Example

# Enable TC/TCN packet suppression.
<HUAWEI> system-view
[~HUAWEI] interface 100GE 1/0/1
[*HUAWEI-100GE1/0/1] stp tc-restriction enable
