display bgp routing-table peer

Function

The display bgp routing-table peer command displays the BGP routing information of a specified peer.

Format

Views

All views

Default Level

1: Monitoring level

Usage Guidelines

Usage Scenario

The display bgp routing-table peer command displays the BGP routing information of a specified peer. You can view only specific routing information by specifying different parameters.

Precautions

This command does not display the default route that is configured to be advertised to peers using the peer default-route-advertise command.

By default, if accepted-routes or received-routes is specified, the routes that match the configured import policy are displayed. If not-accepted-routes is specified, no information is displayed. After the keep-all-routes or peer keep-all-routes command is run to configure the device to save all BGP routing updates from a specified peer after a BGP connection is established:

  • If received-routes is specified, all routes received from peers are displayed.
  • If accepted-routes is specified, the routes that match the configured import policy are displayed.
  • If not-accepted-routes is specified, the routes that do not match the configured import policy are displayed.
