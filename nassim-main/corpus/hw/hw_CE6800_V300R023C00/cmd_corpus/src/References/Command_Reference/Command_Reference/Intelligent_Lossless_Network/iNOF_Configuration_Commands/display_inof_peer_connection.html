display inof peer connection

Function

The display inof peer connection command displays information about the iNOF connection established between the local and peer devices after the iNOF reflector and client are configured.

This command is supported only on the CE6860-SAN, CE6866K, CE6866, CE6860-HAM, CE6885-SAN, CE8850-SAN, CE8851K, CE8851-32CQ8DQ-P and CE8850-HAM.

Format

display inof peer connection

Parameters

None

Views

All views

Default Level

1: Monitoring level

Usage Guidelines

After the iNOF reflector and client are configured, you can run this command to check information about the iNOF connection established between the local and peer devices, including the local role, local address, and peer address.

Table 1 Description of the display inof peer connection command output
Item Description
IPv4 Info

IPv4 information.
Index

Index.
PeerIP

IP address of the peer device.
ConnectStatus

Connection status. The options are as follows:

  • Established: indicates that the connection is successful.
  • Connect: indicates that the connection is in progress.
  • OpenSend: indicates that the local end has sent an Open message and waits for a response from the peer end. If no response is received within the timeout period, the status changes to Connect.
  • OpenConfirm: indicates that the local end starts to send keepalive packets and waits for a response from the peer end.
EstablishTime

Connection establishment time.

IPv6 information.
Role

Device role.
Port

Port ID.
ServiceAddress
