port discard tagged-packet

Function

The port discard tagged-packet command enables an interface to discard incoming tagged packets.

The undo port discard tagged-packet command disables an interface from discarding incoming tagged packets.

By default, an interface does not discard incoming tagged packets.

Format

port discard tagged-packet

undo port discard tagged-packet

Parameters

None

Views

Layer 2 100GE interface view,Layer 2 10GE interface view,Layer 2 200GE interface view,25GE-L2 view,400GE-L2 view,Layer 2 50GE interface view,Layer 2 Eth-Trunk interface view,Layer 2 GE interface view,Interface group view

Default Level

2: Configuration level

Usage Guidelines

Usage Scenario

Network hosts send untagged packets. When an interface on a switching device connects to hosts, this interface needs only to process untagged packets. To enable the interface to discard incoming tagged packets, run the port discard tagged-packet command. This command prevents hosts from accessing other switching devices after the interface function is changed without permission.

Precautions

The port discard tagged-packet command does not take effect on Dot1q-tunnel interfaces.

The port discard tagged-packet command can be configured only on interfaces connecting to hosts. If this command is configured on interfaces connecting to other network devices, VLAN users of these interfaces cannot communicate with each other.

Example

# Enable 100GE1/0/1 to discard incoming tagged packets.
<HUAWEI> system-view
[~HUAWEI] interface 100GE 1/0/1
[~HUAWEI] portswitch
[*HUAWEI-100GE1/0/1] port discard tagged-packet
