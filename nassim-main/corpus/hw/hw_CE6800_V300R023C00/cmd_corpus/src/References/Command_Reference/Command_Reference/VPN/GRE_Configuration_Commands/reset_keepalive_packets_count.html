reset keepalive packets count

Function

The reset keepalive packets count command clears the statistics about the Keepalive messages and Keepalive response messages sent and received by the local GRE tunnel interface.

This command is supported only on the CE6863H, CE6863H-K, CE6860-SAN, CE6866K, CE6866, CE6860-HAM, CE6855-48XS8CQ, CE6885-SAN, CE8850-SAN, CE8855, CE8851-32CQ4BQ, CE8851K, CE8851-32CQ8DQ-P, CE8850-HAM, CE6881H, CE6881H-K, CE6820H, CE6820H-K, CE6820S, CE6885, CE6885-T, CE6885-LL (standard forwarding mode) and CE6863E-48S8CQ.

Format

reset keepalive packets count

Parameters

None

Views

Tunnel interface view

Default Level

2: Configuration level

Usage Guidelines

Usage Scenario

Before configuring a tunnel policy to select GRE tunnels as VPN tunnels, enable the Keepalive function for GRE tunnels to prevent a VPN from selecting an unreachable GRE tunnel as its tunnel. After configuring the Keepalive function, you can run the display keepalive packets count command to check the numbers of Keepalive messages and Keepalive response messages sent and received by a GRE tunnel interface.

The reset keepalive packets count command enables a device to re-collect statistics about Keepalive messages and Keepalive response messages, facilitating monitoring on the operating status of GRE tunnels.

Prerequisites

This command takes effect only on a GRE tunnel interface.

Example

# Clear statistics about the Keepalive messages and Keepalive response messages sent and received by the local GRE tunnel interface Tunnel 100.
<HUAWEI> system-view
[~HUAWEI] interface tunnel 100
[*HUAWEI-Tunnel100] tunnel-protocol gre
[*HUAWEI-Tunnel100] commit
[~HUAWEI-Tunnel100] reset keepalive packets count
