The regular expression in the JSON script uses the POSIX syntax.

If the regular expression is written using the POSIX syntax in the user script, EVA uses the POSIX regular expression for matching.

If another syntax is used to write a regular expression, EVA attempts to translate the regular expression for matching. In this case, the filtering result cannot be guaranteed and the accuracy of the translated regular expression cannot be guaranteed.

The rules for EVA to obtain the matching information is as follows:

1. The matching information is stored as a character string array.

2. All results based on the overall matching of the regular expression can be obtained.

3. If the regular expression contains partial matching information, the matching results based on the overall matching are grouped. The first element in each group is the overall matching result, and the matching sequence is from outside to inside and from left to right. Each group stores a maximum of the first 10 matching results as members of the character string array.
