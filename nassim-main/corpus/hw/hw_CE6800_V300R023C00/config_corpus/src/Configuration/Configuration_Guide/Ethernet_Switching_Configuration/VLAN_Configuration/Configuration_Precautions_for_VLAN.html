When using VLAN1, pay attention to the following points:

- You are not advised to use VLAN 1 as the management VLAN or service VLAN.

- Remove the interfaces that do not need to be added to VLAN 1 from VLAN 1 to prevent loops. A trunk interface often needs to permit packets from VLAN 1. Otherwise, some protocol packets transmitted in VLAN 1 may be incorrectly discarded by a trunk interface, causing faults. To prevent such issues, take appropriate measures to avoid potential risks when packets of VLAN 1 are allowed to pass through.

- You are advised to remove interfaces from VLAN 1 on an Eth-Trunk or ring network.

- To prevent broadcast storms in VLAN 1, do not configure the uplink interfaces of access devices to transparently transmit packets from VLAN 1.

- When an interface is bound to a VLANIF interface for Layer 3 forwarding, you are advised to remove the interface from VLAN 1 to prevent Layer 2 loops in VLAN 1.
