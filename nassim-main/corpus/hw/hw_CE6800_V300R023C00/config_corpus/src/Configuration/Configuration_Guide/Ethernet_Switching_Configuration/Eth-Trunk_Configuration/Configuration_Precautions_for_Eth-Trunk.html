In 1:N scenarios, the maximum number of up member links of an Eth-Trunk in static LACP mode is set to 1. Before the active link becomes down and the standby link becomes the active link, the down status of the original active link is not reported to the Eth-Trunk interface. The down status of the original active link is reported to the Eth-Trunk interface only when either of the following conditions is met. This prevents route recalculation on the entire network when the Eth-Trunk interface goes down.

1. The Down timer times out after a delay of 60 seconds.

2. The standby link fails and cannot go up.

Limitations on dynamic load balancing are as follows:

1. This function takes effect only for known unicast traffic, and does not take effect for broadcast, multicast, or unknown unicast traffic.

2. Only dynamic load balancing between panel ports on the same chip takes effect. Therefore, dynamic load balancing does not take effect in inter-card scenarios.

3. Dynamic load balancing and hash-based outbound interface query cannot be used together.

4. Dynamic load balancing performs congestion quantization based on the bandwidth utilization of each panel port in increments of 12.5% of the port bandwidth. The congestion level is the same for panel ports whose bandwidth utilization falls into the same range.

5. For BUM traffic, the device replicates packets and prunes packets that support sticky load balancing. The discarded packets are still included in dynamic load balancing. According to the result, the discarded packets affect the dynamic load balancing effect.

6. Dynamic load balancing and elastic hashing are mutually exclusive.

BUM traffic flows are copied to all Eth-Trunk member interfaces, and then the Eth-Trunk member interfaces that do not need to send the flows are pruned. This occupies the forwarding bandwidth of the Eth-Trunk member interfaces, reducing the actual forwarding capability to be lower than expected. In this case, the device performs bandwidth compensation according to the BUM packet length. For example, if BUM packets with a length of 256 bytes are sent from four 100GE interfaces to one Eth-Trunk interface (consisting of two member interfaces), the actual outbound bandwidth is expected to be higher than 90% of the original bandwidth, but it is 50% without compensation and 65% with compensation.

The outbound bandwidths after compensation are as follows based on the BUM packet length:

128 bytes: 70%

148 bytes: 69%

256 bytes: 65%

512 bytes: 62%

1024 bytes: 61%
