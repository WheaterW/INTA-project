Configuring an Interface to Receive Only Hello Messages with Generation IDs

Context

After PIM-SM is enabled on an interface, the device generates a random number as the generation ID of a Hello message. When the device status changes, a new generation ID is generated. If the device finds that a Hello message received from the same PIM neighbor carries a different generation ID, the device determines that the status of the PIM neighbor has changed.

Procedure

  1. Enter the system view.

    system-view

  2. Enter the PIM interface view.

    interface interface-type interface-number

  3. Switch the working mode of the interface from Layer 2 to Layer 3.

    undo portswitch

    Determine whether to perform this step based on the current interface working mode.

  4. Configure the device to accept only Hello messages with generation IDs and reject Hello messages without any generation IDs.

    pim require-genid

  5. Commit the configuration.

    commit
