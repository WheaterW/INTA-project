For the CE6800 Series:

L2PT delivers ACL rules based on the MAC address and protocol number. LACP and 802.3ah have the same MAC address and protocol number but different sub-protocol numbers. When LACP is enabled, 802.3ah packets are incorrectly sent to the CPU. Similarly, when 802.3ah is enabled, LACP packets are incorrectly sent to the CPU.

For the CE6800 Series:

1. The destination MAC addresses of STP, GVRP, and GMRP packets cannot be replaced with the same multicast MAC address.

2. The destination MAC addresses of EFM (802.3ah), LACP, and DLDP packets cannot be replaced with the same multicast MAC address.

3. When Layer 2 protocol tunneling is configured, the following multicast MAC addresses cannot be used to replace the original multicast MAC addresses of Layer 2 protocol packets:

Destination MAC address of BPDUs: 0180-C200-0000 to 0180-C200-002F

MAC addresses of Smart Link packets: 010F-E200-0004

Special multicast MAC addresses: 0100-0CCC-CCCC and 0100-0CCC-CCCD.

Common multicast MAC addresses that have been used on the device
