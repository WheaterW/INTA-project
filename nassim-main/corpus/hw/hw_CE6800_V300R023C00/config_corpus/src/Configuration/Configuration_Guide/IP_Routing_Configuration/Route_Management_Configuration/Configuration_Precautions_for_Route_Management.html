In ECMP networking, when IP/IPv6 link flapping occurs (for example, multiple interfaces are Down), a large number of ECMP member links converge and apply for new ECMP/NHP resources. As a result, a resource alarm is triggered during link flapping. In addition, when ECMP/NHP resources occupied by multiple intermediate states are insufficient, the ECMP/NHP resources are immediately released and applied for by new ECMP member relationships. As a result, routes that use these intermediate states for forwarding are interrupted for a long time. If ECMP resources fail to be applied for or the chip fails to deliver ECMP resources, the ECMP resources are processed as a single hop.

After ECMP load balancing consistency is enabled, the number of member paths on the forwarding plane increases. As a result, the ECMP fast update performance deteriorates.

Switchover:

1. If a single interface is faulty and the interface on the local board is shut down, the switching takes less than 100 ms. The switching performance is irrelevant to the number of route prefixes.

2. Board fault: A single ECMP member port goes Down due to a board fault, the switching takes less than 600 ms. The switching performance is irrelevant to the number of route prefixes.

3. If multiple interfaces are faulty and 10K (K = 1024) routes exist, the average convergence time of each interface is 500 ms.

If the number of routes is less than 10K (K = 1024), the performance is the same as that when 10K (K = 1024) routes exist. If the number of routes is greater than 10K (K = 1024), the performance is in direct proportion to the number of routes.

Switchback:

1. No packet loss occurs on dynamic routes.

2. Static route switchback requires traffic to trigger ARP learning, which may cause packet loss.

Description:

1. After ECMP load balancing consistency is enabled, the convergence performance of the preceding link faults deteriorates. The convergence performance needs to be increased by 100 ms.

2. When a 100GE link is used as a member path for consistent hash forwarding and a single 100GE interface fails, the switchover time is less than 600 ms.
