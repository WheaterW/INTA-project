Traffic classifier

When the if-match any command is used in a traffic policy to match all data packets:

- If the relationship between rules in a traffic classifier is OR and IPv6 ACLs are matched, if-match any takes effect only for the corresponding IPv6 packets.

- If the relationship between rules in a traffic classifier is OR and the traffic classifier defines if-match gre or if-match vxlan based on a specified ACL rule, if-match any takes effect only for the packet types supported by the specified ACL rule.

- If both if-match any and an ARP-based ACL (23000 to 23999) are configured, if-match any takes effect only for ARP packets.

- If if-match any and if-match unicast, if-match unknown-unicast, or if-match outbound-interface are used together, if-match any takes effect only for non-discarded unicast packets and unknown unicast packets.

- If if-match any and if-match nexthop are used together, if-match any takes effect only for non-discarded Layer 3 packets.

- If if-match any and if-match discard are used together, if-match any takes effect only for discarded packets.

- If the relationship between rules in a traffic classifier is AND, if-match any does not take effect.

- if-match rocev2 cannot be used together with if-match vxlan or if-match gre.

(2) If if-match vlan/inner-vlan is configured in a traffic classifier, the number of occupied ACL resources is affected by the VLAN ID range. You can run the display system tcam acl port-division command to query the number of split ACL rules.

(3) Multiple values can be specified for an if-match matching rule in a traffic classifier. For example, if if-match 8021p 2 3 is configured, the number of occupied ACL resources is the same as the number of values.

(4) The AND and OR modes in a traffic policy affect ACL resource usage. In AND mode, all if-match matching rules except if-match acl/ipv6 acl are combined with ACL rules. For example, if if-match acl 3000 and if-match source-mac are configured in a traffic classifier and three rules are configured in ACL 3000 (assuming that there are three rules after splitting), three ACL resources are occupied in AND mode, and four ACL resources are occupied in OR mode. Note that different values of the same if-match matching rule cannot be combined. For example, if if-match acl 3000 and if-match vlan 1 to 2 are configured in a traffic classifier, three rules are configured in ACL 3000 (assuming that there are three rules after splitting), and two rules are occupied after vlan 1 to 2 is split. In AND mode, six (3 x 2 = 6) ACL resources are occupied. (A flow cannot match both VLAN 1 and VLAN 2. Therefore, three rules are generated by combining VLAN 1 and ACL rules, and another three rules are generated by combining VLAN 2 and ACL rules.) In OR mode, five ACL resources are occupied.

As a result, traffic policies may fail to be applied because there are no bank resources even though the chip has available ACL rule resources. In AND mode, six (3 x 2 = 6) ACL resources are occupied. (A flow cannot match both VLAN 1 and VLAN 2. Therefore, three rules are generated by combining VLAN 1 and ACL rules, and another three rules are generated by combining VLAN 2 and ACL rules.) In OR mode, five ACL resources are occupied.
