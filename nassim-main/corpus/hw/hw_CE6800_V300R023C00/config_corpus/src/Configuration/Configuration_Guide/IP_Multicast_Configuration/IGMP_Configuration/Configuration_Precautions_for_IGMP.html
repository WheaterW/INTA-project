IGMPv1 does not support querier election. An IGMPv1 querier is designated by the upper-layer protocol, such as PIM. In this document version, querier election can be implemented only among multicast devices that run the same MLD version on a network segment. You must configure the same IGMP version on all multicast devices on the same network segment to ensure that the devices can work properly.

1. When the multicast source is located on the user side of the M-LAG, ensure that the multicast source is dual-homed to the M-LAG.

2. In addition to the peer-link, a Layer 3 link must be directly connected between the M-LAG master and backup devices. STP does not need to be enabled on the interfaces at both ends of the Layer 3 link.

3. On the M-LAG master and backup devices, PIM-SM and IGMP must be enabled on all VLANIF interfaces that need to run Layer 3 multicast services, and IGMP snooping must be enabled in the corresponding VLANs.

4. If the Layer 3 link between the M-LAG master and backup devices is connected through a VLANIF interface, PIM must be enabled on the VLANIF interface, and the VLAN corresponding to the VLANIF interface must not be allowed on the peer-link interface.

5. If the peer-link is selected as the optimal link to the RP or multicast source, multicast traffic with the peer-link interface as the outbound interface may fail to be forwarded. To prevent this problem, ensure that the route cost of the Layer 3 link between the M-LAG master and backup devices is less than or equal to the route cost of the peer-link, so that the Layer 3 link between the M-LAG master and backup devices is selected during unicast route selection.
