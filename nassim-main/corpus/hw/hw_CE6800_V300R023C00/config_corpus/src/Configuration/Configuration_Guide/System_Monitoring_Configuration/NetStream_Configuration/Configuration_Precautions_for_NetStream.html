For the CE6800 Series:

Inbound sampling is performed based on original packets. If the forwarding behavior or forwarded packets are modified, the modified information cannot be displayed in the NetStream statistics.

Outbound sampling is performed based on modified packets. If the forwarding behavior or forwarded packets are modified, the modified information can be displayed in the NetStream statistics.

1. The device performs inbound NetStream sampling for the packets on which CAR defined in inbound MQC is taken.

2. The device does not perform inbound NetStream sampling for the packets on which the deny action defined in inbound MQC is taken.

3. The device does not perform inbound NetStream sampling for the packets that are forwarded by the same chip and on which the deny action defined in outbound MQC is taken.

When both NetStream and QoS CAR are configured on a device, the device performs inbound NetStream sampling for the packets rate-limited by CAR.

When NetStream, CPU attack defense, and attack source tracing are configured on a device, the device performs inbound NetStream sampling for the packets rate-limited by CPCAR.

Flow Sampling

Limitations when flow creation is enabled for VXLAN packets:

1. For VXLAN packets whose inner packets are IPv4 packets, VXLAN flexible flows can be created only based on the inner IPv4 packets.

2. For VXLAN packets whose inner packets are not IPv4 packets, VXLAN flexible flows cannot be created and the VXLAN packets are processed as common packets.

3. When VXLAN flexible flow creation is enabled on an interface, the interface no longer creates original flows.

Limitations when flow creation is disabled for VXLAN packets:

The outer packets of VXLAN packets are processed as common packets and flows cannot be created based on the inner packets.
