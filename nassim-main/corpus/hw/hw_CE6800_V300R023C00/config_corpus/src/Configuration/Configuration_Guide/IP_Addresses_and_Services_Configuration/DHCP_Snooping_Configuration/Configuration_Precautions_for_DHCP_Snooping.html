DHCPv4 Snooping

When DHCPv4 snooping is configured in a VXLAN scenario, users need to access the network through VLANs on the access side, but cannot access the network through Layer 2 sub-interfaces or tunnels on the access side. The DHCP server needs to be deployed on the VLAN access side and tunnel side, but not on the Layer 2 sub-interface access side.

1. When users go online from the VLAN access side and the DHCP server is deployed on the VLAN access side, run the dhcp snooping enable command in the BD from which users go online, in the VLAN accessed by users, or on the interface accessed by users, and run the dhcp snooping trusted command on the interface connected to the DHCP server.

2. When users go online from the access side and the DHCP server is deployed on the tunnel side, run the dhcp snooping enable command in the BD from which users go online, in the VLAN accessed by users, or on the interface accessed by users, and run the dhcp snooping trusted tunnel command in the BD to which the DHCP server belongs.

3. When users go online from the access side, the DHCP server is deployed on the tunnel side, and the VBDIF interface on the VTEP accessed by users functions as the DHCP relay agent or DHCP server, run the dhcp snooping enable command in the BD from which users go online, in the VLAN accessed by users, or on the interface accessed by users. You do not need to run the dhcp snooping trusted tunnel command.

DHCPv4 Snooping

When processing packets, DHCP snooping specifies an interface to send Layer 2 packets. In this case, DHCP snooping does not check the source MAC address and static MAC address table.

Scenario description: In a MUX VLAN scenario, a MUX MAC address entry is generated in the principal VLAN for the learned MAC address. During packet forwarding, the source MAC address is verified. In a networking error or exception scenario, packets with the same source MAC address enter the switch through two interfaces at the same time, as a result, the source MAC address of packets fails to be verified, and packets in one direction are discarded. In this scenario, DHCP snooping does not verify packets when forwarding packets based on the service logic. As a result, DHCP packets in both directions are forwarded, which is different from Layer 2 forwarding by hardware.
