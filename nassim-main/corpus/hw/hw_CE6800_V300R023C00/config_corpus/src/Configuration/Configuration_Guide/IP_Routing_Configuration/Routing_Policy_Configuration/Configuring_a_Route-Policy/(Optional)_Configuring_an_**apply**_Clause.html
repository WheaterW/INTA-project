(Optional) Configuring an apply Clause

Procedure

  1. Enter the system view.

    system-view

  2. Enter the route-policy view.

    route-policy route-policy-name matchMode node node

  3. Configure apply clauses in the route-policy. See Table 1.

    Table 1 apply clauses in a route-policy

    Operation

    Command

    Set a cost for routes.

    apply cost { [ + | - ] cost | inherit | none }

    Set the AS_Path attribute.

    apply as-path as-path-value &<1-128> { additive | overwrite | delete }

    apply as-path asValues { additive | overwrite | delete }

    Clear the original AS_Path attribute.

    apply as-path none overwrite

    Set the number of times the most recent AS number in AS_Path is appended to routes.

    apply as-path most-recent most-recent-value

    Set a cost type for routes.

    apply cost-type { external | internal | type-1 | type-2 | internal-inc-ibgp | med-plus-igp }

    Set dampening parameters for EBGP routes.

    apply dampening half-life-reach reuse suppress ceiling

    Delete the BGP route community attribute of a specified value that is set in the community attribute filter.

    apply comm-filter { basIndex | advIndex } delete

    Set BGP community attributes.

    apply community { cmntyValue | cmntyNum | internet | no-advertise | no-export | no-export-subconfed } &<1-32> [ additive ]

    apply community community-list community-list-name [ additive ]

    Clear the existing extended community attributes of routes.

    apply extcommunity rt none

    Set the origin for BGP routes.

    apply origin { egp { egpVal } | igp | incomplete }

    Set the Local_Pref for BGP routes.

    apply local-preference [ + | - ] preference

    Set a BGP Large-community attribute.

    apply large-community-list large-community-list-name { additive | overwrite | delete }

    apply large-community { aa:bb:cc } &<1-16> { additive | overwrite | delete }

    Set the VPN target extended community attribute for BGP routes.

    apply extcommunity { rt extCmntyValue } &<1-16> [ additive ]

    Set one or multiple SoO extended community attributes for BGP routes.

    apply extcommunity soo { site-of-origin } &<1-16> additive

    Set a bandwidth extended community attribute for BGP Routes.

    apply extcommunity bandwidth { extCmntyString | none }

    apply extcommunity bandwidth aggregate [ limit bandwidth-value ]

    apply extcommunity bandwidth aggregate-upstream [ limit upstream-bandwidth ]

    Set a preferred value for BGP routes.

    apply preferred-value preferredVal

    Set a local QoS ID.

    apply qos-local-id qos-local-id

    Set a next-hop address for IPv4 routes.

    apply ip-address next-hop { address | peer-address | blackhole }

    Set a next-hop address for IPv6 routes.

    apply ipv6 next-hop { address | peer-address | blackhole }

    NOTE:

    The CE6885-LL does not support this command.

    Set a level for IS-IS routes.

    apply isis { level-1 | level-1-2 | level-2 }

    Set a preference value for a routing protocol.

    apply preference preference

    Set a tag value for routes.

    apply tag tag

    Set an IP gateway address for routes.

    apply gateway-ip { origin-nexthop | address }

    Set an IPv6 gateway address for routes.

    apply ipv6 gateway-ip { origin-nexthop | address }

    NOTE:

    The CE6885-LL does not support this command.

    The commands in Step 3 can be run in any required order. A node can contain multiple or no apply clauses.

  4. Commit the configuration.

    commit
