- The mirrored outgoing packets may be different from the original packets because the mirroring operation is performed before other forwarding operations on the original packets. For example, if the DSCP value of the original packets needs to be changed, the mirrored packets are different from the original packets because they have been mirrored to the observing port before the change.

- Outbound port mirroring is implemented through internal loopback interfaces. Enabling this function may degrade the forwarding performance of cards.

- When both port mirroring and traffic shaping are configured on an interface, the mirrored packets still contain the packets that are discarded due to traffic shaping.
