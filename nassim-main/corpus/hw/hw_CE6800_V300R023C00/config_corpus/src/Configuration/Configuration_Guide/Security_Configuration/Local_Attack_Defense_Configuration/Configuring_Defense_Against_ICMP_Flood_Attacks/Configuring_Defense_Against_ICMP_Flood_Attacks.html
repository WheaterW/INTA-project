Configuring Defense Against ICMP Flood Attacks

Context

With defense against ICMP flood attacks enabled, the device rate-limits the received ICMP messages, discarding any that exceeds the limit.

Procedure

  1. Enter the system view.

    system-view

  2. Enable defense against ICMP flood attacks.

    anti-attack icmp-flood enable

    You can also run the anti-attack enable command in the system view to enable attack defense against all attack packets, including ICMP flood attack packets.

  3. Configure the rate limit for ICMP flood attack packets.

    anti-attack icmp-flood car cir cir-num

  4. Commit the configuration.

    commit

Verifying the Configuration

Run the display anti-attack statistics icmp-flood command to check statistics relating to defense against ICMP flood attacks.
