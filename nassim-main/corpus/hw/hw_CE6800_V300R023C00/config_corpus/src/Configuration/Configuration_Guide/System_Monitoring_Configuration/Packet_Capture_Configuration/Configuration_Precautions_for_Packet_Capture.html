For the CE6800 Series:

Limitations on obtaining VXLAN packets

For the CE6820H-48S6CQ/CE6820S-48S6CQ/CE6820H-48S6CQ-K/CE6863H-48S6CQ/CE6863H-48S6CQ-K/CE6881H-48S6CQ/CE6881H-48S6CQ-K/CE6881H-48T6CQ/CE6881H-48T6CQ-K:

- If the VXLAN field is not configured, outer packets are matched.

- If the VXLAN field is configured, packets cannot be matched based on the VLAN and destination MAC address.

- If the VXLAN and VNI fields are configured, IPv6 ACLs can be matched based on the leftmost 64 bits of the IPv6 address.

For the CE6855-48XS8CQ/CE6860-HAM/CE6860-SAN/CE6866-48S8CQ-K/CE6866-48S8CQ-P/CE6885-48YS8CQ/CE6863E-48S8CQ/CE6885-LL-56F (standard forwarding mode)/CE6885-48YS8CQ-T/CE6885-SAN-56F:

- For VXLAN packets, inner packets are matched.

- If the VXLAN field is configured, IPv6 ACLs can be matched based on the leftmost 64 bits of the IPv6 address.

For the CE8800 Series:

Limitations on obtaining VXLAN packets

- For VXLAN packets, inner packets are matched.

- If the VXLAN field is configured, IPv6 ACLs can be matched based on the leftmost 64 bits of the IPv6 address.
