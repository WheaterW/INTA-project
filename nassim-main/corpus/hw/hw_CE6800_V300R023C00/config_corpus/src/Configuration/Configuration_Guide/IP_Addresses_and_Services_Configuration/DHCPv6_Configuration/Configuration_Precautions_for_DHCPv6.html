DHCPv6 common

The CAR capability depends on the actual number of packets received by the local device per second.

When the rate of DHCP service packets exceeds the CAR limit, packet loss occurs. The normal packet rate is the same as the packet sending rate. However, the service networking has the following special scenario: The same packet reaches the same device through different paths (the rate of received packets is a multiple of the packet sending rate). In the typical client-relay-server networking, packets from the client can reach the server through the relay agent at Layer 3 or Layer 2. In this case, the server receives two copies of the same packets. In this case, if the CAR capability of the server is verified based on the rate at which the client sends packets. As a result, the CAR limit is exceeded.
