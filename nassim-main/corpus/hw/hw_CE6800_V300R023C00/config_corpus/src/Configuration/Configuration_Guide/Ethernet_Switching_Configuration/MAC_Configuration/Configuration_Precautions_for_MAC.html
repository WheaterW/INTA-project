After MAC address learning limit is enabled on an inter-card Eth-Trunk, only MAC address learning on the local card is limited. When the number of learned MAC addresses reaches the limit and an alarm is generated, the alarm is cleared if the number of learned MAC addresses on a card falls below the limit. If the number of learned MAC addresses on other cards reaches the limit, an alarm is generated again based on the 20-minute timer. After VLAN-based MAC address learning limit is enabled on an inter-card Eth-Trunk and an alarm is generated when the limit is reached, if the outbound interfaces of MAC address entries learned by different cards are physical interfaces, the alarm is cleared when MAC addresses are deleted based on physical interfaces. After 20 minutes, an alarm is reported again if the number of learned MAC addresses on a card reaches the limit.

1. MAC address flapping detection takes effect only in a single ring. In a multi-ring scenario, this function takes effect only in the first ring where MAC address flapping occurs. Specially, if two or more rings exist in a VLAN where MAC address flapping is detected, only information about the interfaces in the first ring where MAC address flapping occurs is displayed in a MAC address flapping alarm, regardless of whether these interfaces are up or down.

2. A device with MAC address flapping detection enabled can detect MAC address flapping only in the first ring where MAC address flapping occurs in a VLAN within a set aging period (configurable, 5 minutes by default). For example, MAC address flapping occurs on PortA and PortB in a VLAN. After either PortA or PortB is set to down, MAC address flapping occurs on PortC and PortD within the same aging period. In this case, a MAC address flapping alarm is generated, indicating that MAC address flapping occurs between PortA and PortB.

3. When MAC address flapping occurs and the loop is not removed, the values in the Original-Port and Move-Ports fields remain unchanged in MAC address flapping records, if the interfaces where MAC address flapping occurs are added to or removed from an Eth-Trunk interface. Delete the MAC address entry that contains the move interface after the loop is removed, and perform MAC address flapping detection again. This facilitates detection on original and move interfaces, and ensures correct loop location and delivery of punishment actions (such as error-down and storm suppression on interfaces) to move interfaces.

4. When frequent switchovers occur on the ring network, the number of MAC address moves may exceed the specified value within a specified period. As a result, the mac-address flapping detection alarm is triggered. In this case, you are advised to lower the MAC address flapping detection security level or prolong the revertive switching time on the ring network.
