Configuring BGP4+ Hierarchical Routing

Context

In Figure 1, DeviceA imports a large number of external routes. To ensure fast route convergence in the case of a device or link failure, BGP4+ hierarchical routing can be deployed on DeviceA, and hierarchical routing convergence can be deployed on other nodes.

Figure 1 Fault scenario

Prerequisites

Before configuring BGP4+ hierarchical routing, you have completed the following task:

  • Configure basic BGP4+ functions.

Procedure

  • Configure DeviceA.
    1. Enter the system view.

      system-view

    2. Create a route-policy node and enter the route-policy view.

      route-policy route-policy-name matchMode node node

    3. Exit the route-policy view.

      quit

    4. Enter the BGP view.

      bgp as-number

    5. Enter the IPv6 unicast address family view.

      ipv6-family unicast

    6. Configure base and hierarchical routes.

      hierarchy-convergence base-route ipv6-address ipv6-mask-length hierarchy-route { all | route-policy policy-name }

      Base routes and hierarchical routes must be BGP4+ routes.

    7. Commit the configuration.

      commit

  • Configure Spine11.
    1. Enter the system view.

      system-view

    2. Enter the BGP view.

      bgp as-number

    3. Enter the IPv6 unicast address family view.

      ipv6-family unicast

    4. Enable the hierarchical routing convergence function.

      hierarchy-convergence enable

    5. Commit the configuration.

      commit

Verifying the Configuration

After completing the configuration, perform the following operations to verify it:

  • Run the display bgp ipv6 routing-table ipv6-address [ mask-length ] command on the peer end to check the BGP IPv6 routing table.
