Limitations on redirection to an interface:

- A traffic policy defining redirection to an interface can be applied only in the inbound direction.

- After packets are redirected to a physical interface, the packets after encapsulation are forwarded out.

- The outbound interface must be in the corresponding VLAN so that traffic can be forwarded.

- Redirection to an interface and deny cannot be configured in the same traffic behavior.

- You are advised to configure redirection to an interface for common Layer 2 forwarding and redirection to the next hop for common Layer 3 forwarding or scenarios where unicast packets are forwarded to a VXLAN tunnel.
