Setting a Dead Interval for OSPFv3 Neighbors

Prerequisites

Before setting a Dead interval for OSPFv3 neighbors, you have completed the following task:

  • Configure basic OSPFv3 functions.

Procedure

  1. Enter the system view.

    system-view

  2. Enter the interface view.

    interface interface-type interface-number

  3. Switch the interface working mode to Layer 3.

    undo portswitch

    Determine whether to perform this step based on the current interface working mode.

  4. Set a Dead interval for OSPFv3 neighbors.

    ospfv3 timer dead interval [ instance instance-id ]

    By default, the Dead interval is four times the length of the Hello interval on the same interface.

    A Dead interval that is shorter than 10 seconds may disconnect the involved OSPFv3 neighbor relationship. To prevent this issue, a minimum of 10 seconds takes effect if the value of dead interval is less than 10 seconds. To ensure that a Dead interval shorter than 10 seconds takes effect, enable the conservative mode by specifying conservative in the ospfv3 timer hello command.

  5. Commit the configuration.

    commit

Verifying the Configuration

Run the display ospfv3 [ process-id ] interface [ area area-id ] [ interface-type interface-number ] command to check OSPFv3 interface information. The Dead field in the command output indicates the Dead interval for OSPFv3 neighbors.
